# PROJECT.md — 『ヨルノハコニワ 〜ひかりの盤と81のかけら〜』
最終更新: 2026-07-09 / **現在位置: Phase 1-b 実装完了・実機確認待ち(H-1〜H-9)**

## 1. プロジェクト概要
- **ジャンル**: 探索型RPG × 将棋学習 × 冒険アドベンチャー(問題集ではなく探索RPG。体感時間 探索6:将棋4)
- **対象**: 小学1〜6年生、将棋初心者〜三段
- **ボリューム**: メイン20〜30h / 全問題80〜120h / コンプ150h+(毎日20〜30分で半年以上)
- **品質目標**: Switch/Steamで販売できる水準。「ゲームとして面白い」が最優先、その結果として将棋が強くなる
- **配信**: GitHub Pages(Web)→ 将来デスクトップ/コンシューマ移植を見据えパッド対応・60fps・解像度非依存
- **アーキテクチャ**: 教育RPGエンジン「HAKONIWA Engine」(コア/教科モジュール/タイトルデータの3層)。将棋は第1教科で、将来英語・算数等へ差し替え可能

## 2. 世界観(完全オリジナル・既存IP禁止)
空の『ひかりの盤』(9×9)から81個の光のかけらが散り、世界は永遠の夜に。見習い棋士の主人公が、ぬいぐるみの王将**ギョク**と共に、将棋の力(=この世界の魔法)でヨドミを浄化し光を取り戻す。トーンは「不気味だけど可愛い」。恐怖・流血・残酷描写は禁止。敵は倒さず浄化して元の姿に戻す。
主要エリア15(村・草原・森・学園・工場・洞窟・図書館・雪原・砂漠・古城・火山・地下遺跡・天空城・ラスボス城・裏)。ボスは八つの「くらやみ大駒」、ラスボスは「うらギョク」。

## 4. Phase 0 成果(実機実証済み)
- エンジン: 反則50件・perft d1〜d5完全一致(30/900/25470/719731/19861490)。実機 d3=38ms, d4=344ms
- 詰み判定: 1手詰=別解全列挙(実機最大1ms)、3手詰=`mateWithin`方式(実機目標200ms以下)。**相互検証で人手の見落とし計4件を機械が検出**(→機械検証必須の規範)
- テスト資産: **T1〜T8=127件**(Node常設ゲート。T8=SubjectModule判定層+収録問題の機械検証)
- iOS: キャッシュ対策(ビルドID全値照合+リトライ)、タッチ安定(Pointer Events一本化)、tests/404のフォールバック
- セーブ: 500KB/1MB可・5MB不可、書き出し3方式成立、**Safari/PWA保存領域分離を実証**、7日規則の長期観察マーカー稼働中
- 描画: タイル+NPC10+粒100で3分60fps

## 5. 技術方針(Phase 0決定・全実装が従う)
①盤=DOM/フィールド=Canvas 2D ②エンジンはDOM非依存UMD・逆探査王手判定・理由コード7種(NIFU/DEAD_PIECE/UCHIFU_ZUME/KING_IN_CHECK/BAD_PROMOTION/BAD_MOVEMENT/BAD_DROP) ③詰みはmateWithin+全別解列挙+無駄合い実用ルール ④キャッシュ=?v+全値照合+no-store+2段フォールバック、**IDはビルドスクリプトで自動刻印** ⑤セーブ=localStorage+schemaVersion+CRC32+書き出し必須・目標500KB ⑥PWA/Safari分離への導線固定+プライベート警告 ⑦週1書き出し促し ⑧問題は収録前機械検証(成/不成・全打点)
